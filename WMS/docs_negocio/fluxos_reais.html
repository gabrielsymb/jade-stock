Fluxos Operacionais

Fluxos Reais do Dia a Dia (visão de produto)

Esta página mostra como cada operação funciona ponta a ponta: ação do operador, validações, atualização de saldo, histórico e evento.

Fluxo A: Movimentação de Estoque

Operador informa dados -> API /v1/movimentacoes -> valida SKU/endereço/saldo -> atualiza saldo -> grava movimentação -> grava evento
  • Endpoint: POST /v1/movimentacoes
  • Caso de uso: registrar_movimentacao_estoque.py
  • Teste: test_registrar_movimentacao_estoque.py
  • Banco: movimentacao_estoque + saldo_estoque

Fluxo B: Ajuste de Estoque

Operador informa ajuste + motivo -> API /v1/ajustes -> valida regra -> corrige saldo -> grava ajuste -> grava evento
  • Endpoint: POST /v1/ajustes
  • Caso de uso: registrar_ajuste_estoque.py
  • Teste: test_registrar_ajuste_estoque.py
  • Banco: movimentacao_estoque + saldo_estoque

Fluxo C: Recebimento com Conferência

Operador informa nota + itens -> API /v1/recebimentos -> compara esperado x conferido -> atualiza saldo -> grava recebimento -> grava eventos
  • Endpoint: POST /v1/recebimentos
  • Caso de uso: registrar_recebimento.py
  • Teste: test_registrar_recebimento.py
  • Banco: recebimento + recebimento_item + saldo_estoque

Observação Importante

  • WMS_API_BACKEND=inmemory: fluxo roda para desenvolvimento sem gravar banco real.
  • WMS_API_BACKEND=postgres: fluxo persiste no banco PostgreSQL.
  • Transação ativa: se uma etapa falhar, o sistema desfaz tudo.

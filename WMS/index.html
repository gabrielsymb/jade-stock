Como o Sistema Trabalha (fluxo visual)

Regra de Negócio -> Caso de Uso -> Validação -> Persistência -> Evento -> API/Integração

Exemplo prático: o PDV chama POST /v1/movimentacoes, o sistema valida as regras, atualiza saldo, grava histórico e gera evento.

API Sem Jargão

Endpoint é um "botão remoto" de ação de negócio. Em vez de outro sistema acessar banco direto, ele chama esses botões:

  • POST /v1/movimentacoes: entrada, saída e transferência
  • POST /v1/ajustes: correção de estoque com motivo
  • POST /v1/avarias: baixa por perda operacional com motivo obrigatório
  • POST /v1/recebimentos: entrada com conferência
  • POST /v1/inventarios/ciclico: contagem cíclica com ajuste automático
  • POST /v1/kanban/politicas: definição de faixas e acionamento de reposição
  • POST /v1/curva-abcd/processar: classificação ABCD e atualização de política de cobertura
  • POST /v1/giro/processar: cálculo de giro/cobertura e alertas operacionais
  • POST /v1/sazonalidade/processar: aplicação de sinal sazonal externo com regras determinísticas
  • POST /v1/orcamento/simular: validação financeira da compra com exceções auditáveis

Conceitos Técnicos Traduzidos

  • FastAPI: ferramenta para abrir a "porta" da integração
  • Uvicorn: processo que mantém essa porta ligada
  • Validação: bloqueio de operação inválida antes de gravar
  • Idempotência: evitar processar a mesma operação duas vezes
  • Transação: ou grava tudo, ou cancela tudo

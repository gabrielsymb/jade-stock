Tecnologia em Linguagem de Negócio

Decisões Técnicas Traduzidas

Esta página explica por que certas decisões foram tomadas no sistema, sem exigir conhecimento de programação.

1) FastAPI e Uvicorn

  • FastAPI: ferramenta para criar os endpoints do sistema.
  • Uvicorn: processo que mantém a API funcionando em uma porta.
  • Benefício de negócio: integração com PDV e outros sistemas sem acesso direto ao banco.
  • Arquivo: wms/interfaces/api/app.py

2) Validação antes de gravar

Recebe pedido -> valida regra de negócio -> só então grava
  • Exemplo: não movimenta sem saldo suficiente.
  • Exemplo: não ajusta sem motivo.
  • Benefício: reduz erro operacional e financeiro.
  • Arquivos: registrar_movimentacao_estoque.py, registrar_ajuste_estoque.py

3) Idempotência (evitar duplicidade)

Mesmo pedido repetido -> sistema não deve duplicar efeito
  • No recebimento, já existe proteção por nota_fiscal + correlation_id.
  • Benefício: evita processar duas vezes a mesma entrada.
  • Arquivo: registrar_recebimento.py
  • Tabela: recebimento (unique de nota+corr)

4) Transação e rollback

Atualiza saldo + grava histórico + grava evento -> se falhar algo, desfaz tudo
  • Benefício: não deixa o sistema “meio certo, meio errado”.
  • Arquivo: postgres_transaction_manager.py
  • Teste: test_postgres_transactional_behavior.py

5) Concorrência (duas operações ao mesmo tempo)

Duas requisições simultâneas -> sistema protege saldo para não corromper
  • Benefício: evita erro de saldo em operação real com múltiplos operadores.
  • Arquivo: postgres_estoque_repository.py
  • Teste: test_postgres_transactional_behavior.py

6) Camadas do sistema

Regra de negócio (o que pode) -> Caso de uso (como executar) -> Infraestrutura (onde grava)
  • Benefício: mudar tecnologia sem quebrar regra do negócio.
  • Pastas: wms/domain, wms/application, wms/infrastructure
